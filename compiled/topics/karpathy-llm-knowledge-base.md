# Summary

Andrej Karpathy introduced a system for personal knowledge management using Large Language Models (LLMs), which he calls the Karpathy LLM Knowledge Base. This system uses an AI-powered approach to compile and maintain a structured wiki of research materials, enabling efficient query resolution without relying on retrieval-augmented generation (RAG) techniques. The setup involves three directories (`raw/`, `wiki/`, and `outputs/`) and is managed through a schema file that guides the LLM's behavior.

# Key Insights

- Karpathy's system leverages an LLM to automatically generate structured knowledge articles from raw research material, maintaining consistency and backlinks between related concepts.
- The core of this approach lies in active maintenance: incremental updates based on new sources and periodic linting passes that ensure structural integrity and factual accuracy.
- The simplicity of the directory structure and Markdown format allows for easy customization and adaptation by other users.
- This method bypasses traditional RAG pipelines, relying instead on the LLM's context window to handle full document indexing without chunking issues.
